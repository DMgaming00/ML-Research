## 1. Core Experiments to Run

### Experiment A: The "Shortcut" Ablation (Color Jitter)
- **Hypothesis:** Without color jitter, the network cheats using color histograms, completely failing to learn semantic structures.
- **Action:** In the `SimCLR_Notebook.ipynb` Step 2, set `p=0` for both `RandomApply([color_jitter])` and `RandomGrayscale`.
- **Expected Outcome:** The NT-Xent training loss will drop to near-zero very quickly (the network thinks it is doing great). However, when evaluated on the downstream linear classification task, the accuracy will be abysmal, and the t-SNE plot will show mixed, unclustered points.

### Experiment B: Batch Size Scaling
- **Hypothesis:** Contrastive learning relies on in-batch negatives. Larger batches = more negatives = a harder contrastive task = better representations.
- **Action:** In Step 2, change the `DataLoader` batch size from `256` to `32`, and then to `512` (if your GPU allows).
- **Expected Outcome:** The linear evaluation accuracy on CIFAR-10 will scale linearly with the log of the batch size. 

### Experiment C: The Role of Temperature ($\tau$)
- **Hypothesis:** Temperature scales the gradients and dictates how much the model penalizes "hard negatives."
- **Action:** In Step 4, change the `temperature` parameter in `NTXentLoss` from `0.5` to `0.1` and `1.0`.
- **Expected Outcome:** $\tau=0.5$ generally performs best. $\tau=1.0$ will result in soft distributions and poor separation. $\tau=0.1$ might cause numerical instability or over-penalize semantically similar negatives.

### Experiment D: Ablating the Projection Head
- **Hypothesis:** Applying the loss directly to the ResNet output $h$ destroys useful downstream information.
- **Action:** In the Linear Evaluation Step, extract `z` instead of `h` to train the logistic regression classifier.
- **Expected Outcome:** The linear evaluation accuracy will plummet, proving that $z$ has become invariant to useful features (like color), while $h$ was protected by the projection head buffer.

## 2. Visualizations Generated by the Notebook

### Plot 1: The Loss Curve
- **Generated in Step 5.**
- **Why it matters:** Contrastive loss can be deceiving. A loss near zero often means the model has found a shortcut (like the color histogram cheat) rather than learning rich semantics. The loss should decrease smoothly and plateau, but rarely hit absolute zero.

### Plot 2: t-SNE Visualization of the Embeddings
- **Generated in Step 6.**
- **Why it matters:** t-SNE reduces the 512-dimensional vector $h$ (from ResNet18) into 2 dimensions. By coloring the points using the ground-truth CIFAR-10 labels (which the model never saw during training), we visually prove the model learned to cluster semantics unsupervised. Clear groupings of the same colors validate the entire pipeline.
